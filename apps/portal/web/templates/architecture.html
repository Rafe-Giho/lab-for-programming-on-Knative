Service Architecture

Code Runner Lab 실행 구조

사용자가 웹에서 코드를 제출하면 포털이 Kubernetes Job을 만들고, 선택한 언어와 버전에 맞는 러너 컨테이너가 격리된 환경에서 코드를 실행한 뒤 결과를 다시 포털로 반환합니다.

Flow

실행 흐름

1 Browser

사용자가 코드와 언어, 버전을 선택해 실행 요청을 보냅니다.

2 Knative Portal

포털이 요청을 검증하고 실행할 런타임 이미지를 결정합니다.

3 Kubernetes Job

요청마다 별도 Job과 Pod를 생성해 실행을 분리합니다.

4 Language Runner

선택한 언어/버전 러너가 코드를 실행하고 로그와 종료코드를 남깁니다.

5 Result Return

포털이 결과를 수집해 사용자 화면에 출력합니다.

Components

구성 요소

Portal Service 화면 제공, 입력 검증, Job 생성 요청, 로그 수집을 담당합니다.
Execution Namespace 실행 Pod를 별도 네임스페이스에서 관리해 권한과 자원을 분리합니다.
Runner Images Python, Java, C, C++용 언어/버전별 이미지가 따로 준비되어 선택한 런타임에 맞춰 실행됩니다.

Isolation

격리 포인트

요청 단위 분리 실행 요청마다 Pod가 따로 생성되어 상태가 섞이지 않습니다.
임시 작업 공간 코드는 Pod 내부 작업 공간에만 기록되고 완료 후 Job TTL에 따라 정리됩니다.
실행 제한 타임아웃, 리소스 제한, non-root 실행 설정을 함께 적용합니다.
